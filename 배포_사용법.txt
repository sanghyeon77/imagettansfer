============================================================
📦 배포용 ZIP 파일 사용 가이드
============================================================

✅ ZIP 파일이 생성되었습니다!

파일명: video_auto_capture_YYYYMMDD_HHMMSS.zip
크기: 약 20KB (영상 파일 제외)

============================================================

📤 다른 컴퓨터에 배포하는 방법

[1단계] ZIP 파일 전송
   → USB 메모리, 이메일, 클라우드 드라이브 등으로 전송
   → 또는 네트워크 공유 폴더 사용

[2단계] 압축 해제
   → 원하는 위치에 압축 해제
   → 예: C:\video_capture\
   → 예: D:\Programs\video_auto_capture\

[3단계] 새 컴퓨터에서 설치
   1. Python 설치 (https://www.python.org/)
      → "Add Python to PATH" 체크 필수!
   
   2. FFmpeg 설치
      → choco install ffmpeg (Chocolatey 사용)
      → 또는 수동 설치
   
   3. install.bat 실행
      → Python 패키지 자동 설치

[4단계] 설정
   1. config.example.json → config.json 복사
   2. config.json 편집:
      - 디스코드 웹훅 URL 입력
      - 영상 파일 경로 설정
   3. 영상 파일을 video1, video2 폴더에 복사

[5단계] 실행
   → python dual_video_to_discord.py
   또는
   → start_auto.bat 더블클릭

============================================================

📋 ZIP 파일에 포함된 내용

✅ 프로그램 파일:
   - dual_video_to_discord.py (메인 프로그램)
   - video1_to_discord.py (단일 영상용)
   - video2_to_discord.py (단일 영상용)

✅ 설정 파일:
   - config.example.json (설정 예시)
   - requirements.txt (Python 패키지 목록)

✅ 설치/실행 스크립트:
   - install.bat (패키지 설치)
   - start_auto.bat (프로그램 실행)
   - stop.bat (프로그램 중지)
   - register_autostart.bat (자동 시작 등록)
   - unregister_autostart.bat (자동 시작 해제)

✅ 문서:
   - README.md (전체 설명)
   - DEPLOYMENT_GUIDE.md (배포 가이드)
   - AUTOSTART_GUIDE.md (자동 시작 가이드)
   - 시작하기.txt (빠른 시작 가이드)
   - 기타 가이드 문서들

✅ 폴더:
   - temp_images/ (임시 이미지 저장)
   - video1/ (첫 번째 영상)
   - video2/ (두 번째 영상)

============================================================

⚠️ 주의사항

1. 영상 파일은 ZIP에 포함되지 않습니다
   → 용량이 크기 때문에 별도로 전송 필요

2. config.json은 포함되지 않습니다
   → 웹훅 URL 보안을 위해 제외
   → config.example.json을 복사하여 사용

3. 압축된 영상 파일도 포함되지 않습니다
   → 새 컴퓨터에서 다시 압축됩니다

🔄 새 버전 배포하기

프로그램을 수정한 후 다시 배포하려면:

2. 새로운 ZIP 파일 생성됨
3. 날짜/시간이 포함된 파일명으로 자동 생성

============================================================

💡 배포 팁

1. 영상 파일 별도 전송
   → 클라우드 드라이브 (Google Drive, OneDrive 등)
   → 외장 하드 또는 USB

2. 설정 파일 공유
   → config.json을 별도로 공유 (보안 주의!)
   → 또는 구두로 웹훅 URL 전달

3. 문서 함께 제공
   → 시작하기.txt 먼저 읽도록 안내
   → DEPLOYMENT_GUIDE.md 참고하도록 안내

4. 테스트 환경 구축
   → 가상 머신에서 먼저 테스트
   → 모든 단계가 정상 작동하는지 확인

GitHub, GitLab 등에 업로드하려면:

2. README.md 업데이트
   → 설치 방법 명확히 작성
   → 스크린샷 추가 (선택)

3. 릴리스 노트 작성
   → 버전 정보
   → 변경 사항
   → 알려진 문제

📞 사용자 지원

배포 후 사용자가 문제를 겪는다면:
